# Ollie Writes Things

A personal blog and portfolio site built with [Astro](https://astro.build/), managed with [TinaCMS](https://tina.io/), and deployed to GitHub Pages.

**Live Site:** https://olliewritesthings.com

## 🚀 Quick Start

### Development

```bash
# Install dependencies
npm install

# Start local dev server
npm run dev
# Accessible at http://localhost:3000/

# Build for production
npm run build

# Preview production build
npm run preview
```

### Admin Access

- **Local Development:** http://localhost:3000/admin/
- **Production:** https://olliewritesthings.com/admin/

## 📝 Content Management

Content is managed through **TinaCMS** in two collections:

### Blog Posts (`src/content/blog/`)
- Auto-managed by TinaCMS at `/admin/`
- Fields: title, description, pubDate, updatedDate, heroImage, body
- Format: Markdown (.md) or MDX (.mdx)

### Pages (`src/content/pages/`)
- Auto-managed by TinaCMS at `/admin/`
- Fields: title, intro, body
- Format: Markdown (.md)

### TinaCMS Setup

To enable full TinaCMS functionality, see [TINACMS_SETUP.md](./TINACMS_SETUP.md) for:
- Setting up TinaCloud account
- Configuring GitHub Actions secrets
- Local development setup
- Troubleshooting

## 🚀 Deployment

The site automatically deploys to GitHub Pages on every push to `main`:

1. GitHub Actions builds Astro site (`npm run build`)
2. TinaCMS admin assets are generated (if credentials configured)
3. Everything is deployed to https://olliewritesthings.com/

**Custom Domain:** olliewritesthings.com (via CNAME record)

## 🔐 Security

- Sensitive credentials (TinaCloud token, client ID) are:
  - **Never hardcoded** in the repository
  - Stored as **GitHub Actions Secrets** for CI/CD
  - Can be set in `.env` locally (which is gitignored)
- The `.env` file is in `.gitignore` for development
